## 二、需求一：仓位合计约束

### 2.1 背景与问题

当前持仓合计理论上不可能超过 100%，但系统在以下三处均没有做合计校验：

**① `approvals.py` `modify_weights`**
只校验单标的不超 `max_position_weight`，不校验全仓合计。

```python
# 现状：只检查单标的
for sym, wt in payload.modified_weights.items():
    if wt > max_pos:  # ← 只防单标的，不防合计
        raise HTTPException(...)
```

**② `portfolio.py` `apply_recommendations`**
审批通过后直接用建议权重覆盖持仓，无合计校验。

**③ `pipeline.py` `_make_recommendations`**
推荐调仓幅度用随机数，完全不感知当前总仓位水位：

```python
# 现状：buy 就加 3%~8%，不管总仓位已经 95% 了
rec = min(cur + random.uniform(0.03, 0.08), 0.20)
```

### 2.2 功能需求

#### 需求 2.2.1 后端——审批修改权重时校验全仓合计

**接口**：`PUT /api/v1/approvals/{approval_id}/modify`

**校验基准说明**：
- "其他持仓权重之和"以 `portfolio_holdings` 表中当前实际权重为基准
- 排除本次 `modified_weights` 中出现的标的后求和
- 即：`other_weight = sum(当前持仓所有标的权重) - sum(modified_weights 中出现标的的当前权重)`
- 校验公式：`other_weight + sum(modified_weights.values()) > 1.0` 则返回 422

**错误响应格式**：
```json
{
  "detail": "修改后全仓合计 108.0% 超过 100%，当前其他持仓已占 93.0%，本次可用余量 7.0%"
}
```

**实现逻辑**：
```python
# approvals.py modify_weights 中，单标的校验之后、更新 recommendations 之前
portfolio_result = await db.execute(select(PortfolioHolding))
current_holdings = {h.symbol: h.weight for h in portfolio_result.scalars().all()}

# 计算不在本次修改范围内的标的权重之和
other_weight = sum(
    w for sym, w in current_holdings.items()
    if sym not in payload.modified_weights
)
new_total = other_weight + sum(payload.modified_weights.values())
if new_total > 1.0 + 0.001:  # 允许 0.1% 浮点误差
    avail = round((1.0 - other_weight) * 100, 1)
    total = round(new_total * 100, 1)
    other = round(other_weight * 100, 1)
    raise HTTPException(status_code=422, detail=
        f"修改后全仓合计 {total}% 超过 100%，当前其他持仓已占 {other}%，本次可用余量 {avail}%")
```

#### 需求 2.2.2 后端——apply_recommendations 落库前预校验

**函数**：`portfolio.py` `apply_recommendations`

在执行任何 UPDATE 前，先模拟计算落库后的全仓合计。若超过 1.0，抛出异常阻止落库。

审批通过和修改权重后通过两条路径都会调用此函数，此处作为兜底防线。

```python
async def apply_recommendations(db, decision_id, approval_id, recommendations):
    if not recommendations:
        return

    # 兜底校验：模拟落库后的合计
    all_holdings_result = await db.execute(select(PortfolioHolding))
    current = {h.symbol: h.weight for h in all_holdings_result.scalars().all()}
    rec_dict = {r["symbol"].upper(): r["recommended_weight"] for r in recommendations}

    simulated = {**current, **rec_dict}  # 合并（rec 覆盖 current）
    # 过滤掉权重 <= 0 的（清仓操作）
    total = sum(w for w in simulated.values() if w > 0.0001)
    if total > 1.0 + 0.001:
        print(f"[portfolio] ⚠ 落库校验失败：合计 {total:.1%}，阻止写入")
        raise ValueError(f"apply_recommendations: 全仓合计 {total:.1%} 超过 100%，已阻止落库")

    # 原有逻辑继续...
```

#### 需求 2.2.3 后端——pipeline 推荐量改为基于可用余量计算

**函数**：`pipeline.py` `_make_recommendations`

用确定性计算替换随机调仓幅度：

```
可用余量 = 1.0 - 其他所有持仓的权重之和（不含当前标的）
buy 推荐权重 = min(当前权重 + min(0.10, 可用余量), max_position_weight)
sell 推荐权重 = max(当前权重 - 0.10, 0.0)
hold 推荐权重 = 当前权重（不变）
```

**满仓降级**：可用余量 ≤ 0.001 时，buy 方向降级为 hold，`low_data_warning` 中追加"当前仓位已满，无法加仓"。

```python
def _make_recommendations(signals, context):
    symbols  = context.get("symbols", [])
    portfolio = context.get("current_portfolio") or {s: 0.0 for s in symbols}
    risk_cfg  = context.get("risk_cfg", {})
    max_pos   = risk_cfg.get("max_position_weight", 0.20)

    # 计算当前全仓总权重
    def get_weight(h):
        return h.get("weight", 0.0) if isinstance(h, dict) else float(h or 0.0)

    total_weight = sum(get_weight(h) for h in portfolio.values())

    # ... 投票逻辑不变 ...

    recs = []
    for sym in symbols:
        h   = portfolio.get(sym, 0.0)
        cur = get_weight(h)

        # 可用余量 = 1.0 - 其他标的权重之和
        other_weight = total_weight - cur
        available    = max(0.0, 1.0 - other_weight)

        if final_dir == "buy":
            if available <= 0.001:
                # 满仓降级
                rec = cur
                low_data_warning = (low_data_warning or "") + " 当前仓位已满，无法加仓"
            else:
                increment = min(0.10, available)
                rec = min(cur + increment, max_pos)
        elif final_dir == "sell":
            rec = max(cur - 0.10, 0.0)
        else:
            rec = cur

        recs.append({
            "symbol":             sym,
            "current_weight":     round(cur, 4),
            "recommended_weight": round(rec, 4),
            "weight_delta":       round(rec - cur, 4),
            "confidence_score":   round(avg_conf, 3),
            "similar_cases":      [],
            "missing_agents":     missing_agents,
            "low_data_warning":   low_data_warning,
        })
    return recs
```

### 2.3 验收标准

1. 持仓合计 93%，审批页修改某标的权重至 10%（合计 103%），后端返回 422，detail 包含"可用余量仅 7%"。
2. 持仓合计 93%，前端输入框 `max` 属性自动限制为 7%（`min(max_position_weight, 0.07)`），无法手动输入超出值。
3. pipeline 在总仓位已满（≥ 99.9%）时，buy 信号的推荐权重等于当前权重（不增加），`low_data_warning` 包含"当前仓位已满"。
4. 持仓合计为 0 时，buy 推荐权重 = `min(当前权重 + 0.10, max_position_weight)`。
5. `apply_recommendations` 兜底校验：若上层校验失效导致合计超限，落库被阻止并打印日志，不写入错误数据。

## 三、需求二：图谱真实联通

### 3.1 背景与问题

当前图谱模块存在 **6 处造假 / 割裂**，导致完全没有实用价值：

| 问题 | 代码位置 | 现状 |
|------|---------|------|
| 收益率是随机数 | `services/graph.py:29` | `outcome_return = random.uniform(-0.08, 0.15)` |

### 3.2 功能需求

#### 需求 3.2.1 图谱节点收益结算（T+5 真实价格）

**定时任务实现方案**：使用 `APScheduler`（`pip install apscheduler`），在 `main.py` 的 `lifespan` 中注册。

```python
# main.py lifespan 中
from apscheduler.schedulers.asyncio import AsyncIOScheduler

@asynccontextmanager
async def lifespan(app: FastAPI):
    await init_db()
    scheduler = AsyncIOScheduler()
    # 每个交易日 16:05 执行（A 股 16:00 收盘，等 5 分钟确保数据可用）
    scheduler.add_job(settle_graph_nodes, "cron", hour=16, minute=5,
                      id="settle_nodes", replace_existing=True)
    scheduler.start()
    yield
    scheduler.shutdown()
```

**交易日判断**：使用 AKShare `ak.tool_trade_date_hist_sina()` 获取交易日历，缓存当年所有交易日，判断当天是否为交易日。

```python
# services/settlement.py
import akshare as ak
from datetime import date, timedelta

_trade_dates_cache: set[str] = set()

def _load_trade_dates():
    global _trade_dates_cache
    if not _trade_dates_cache:
        df = ak.tool_trade_date_hist_sina()
        _trade_dates_cache = set(df["trade_date"].astype(str).tolist())

def is_trade_date(d: date) -> bool:
    _load_trade_dates()
    return d.strftime("%Y-%m-%d") in _trade_dates_cache

def get_nth_trade_date_before(d: date, n: int) -> date:
    """获取 d 之前第 n 个交易日"""
    _load_trade_dates()
    count = 0
    cur = d - timedelta(days=1)
    while count < n:
        if cur.strftime("%Y-%m-%d") in _trade_dates_cache:
            count += 1
        if count < n:
            cur -= timedelta(days=1)
    return cur
```

**结算逻辑**（`services/settlement.py`）：
```python
async def settle_graph_nodes():
    """扫描并结算 T+5 到期的图谱节点"""
    from database import AsyncSessionLocal
    async with AsyncSessionLocal() as db:
        today = date.today()
        if not is_trade_date(today):
            return

        # 找距今恰好 5 个交易日前的节点（未结算、已审批通过）
        five_days_ago = get_nth_trade_date_before(today, 5)
        cutoff_start  = datetime.combine(five_days_ago, time(0, 0))
        cutoff_end    = datetime.combine(five_days_ago, time(23, 59))

        result = await db.execute(
            select(GraphNode).where(
                GraphNode.outcome_settled == False,
                GraphNode.approved == True,
                GraphNode.timestamp >= cutoff_start,
                GraphNode.timestamp <= cutoff_end,
            )
        )
        nodes = result.scalars().all()

        for node in nodes:
            await _settle_one_node(db, node)

        await db.commit()

async def _settle_one_node(db, node: GraphNode):
    try:
        symbol = node.symbols[0] if node.symbols else None
        if not symbol:
            return

        # 拉取当天收盘价
        today_str = date.today().strftime("%Y%m%d")
        snap = await fetch_cn_snapshot(symbol)
        if not snap:
            return

        settle_p  = snap["current_price"]
        approve_p = node.approve_price

        if not approve_p or approve_p <= 0:
            return

        outcome_return = round((settle_p - approve_p) / approve_p, 4)

        # 获取关联的 DecisionRun 的 final_direction
        approval_result = await db.execute(
            select(ApprovalRecord).where(ApprovalRecord.id == node.approval_id)
        )
        approval = approval_result.scalars().first()
        direction = None
        if approval and approval.decision_run_id:
            dr_result = await db.execute(
                select(DecisionRun).where(DecisionRun.id == approval.decision_run_id)
            )
            dr = dr_result.scalars().first()
            if dr:
                direction = dr.final_direction

        direction_correct = None
        if direction == "buy":
            direction_correct = outcome_return > 0
        elif direction == "sell":
            direction_correct = outcome_return < 0
        elif direction == "hold":
            direction_correct = True  # hold 不评判方向

        node.outcome_return            = outcome_return
        node.outcome_sharpe            = _calc_simple_sharpe(outcome_return)
        node.settle_price              = settle_p
        node.outcome_settled           = True
        node.outcome_settled_at        = datetime.now()
        node.outcome_direction_correct = direction_correct

        print(f"[settlement] {symbol} 结算: {outcome_return:+.2%} 方向{'正确' if direction_correct else '错误'}")
    except Exception as e:
        print(f"[settlement] 节点结算失败: {e}")

def _calc_simple_sharpe(daily_return: float, risk_free: float = 0.0002) -> float:
    """单点简化 Sharpe（仅供参考，完整 Sharpe 需要多期数据）"""
    excess = daily_return - risk_free * 5  # 5日无风险收益
    return round(excess / max(abs(daily_return) * 0.5, 0.01), 2)
```

#### 需求 3.2.2 图谱 embedding 改为真实特征向量

用实际信号特征替换 hash+random 的伪 embedding，使相似度检索有实际意义。

**辅助函数**：
```python
def _dir_to_float(sig, direction):
    if not sig: return 0.0
    return 1.0 if sig.get("direction") == direction else 0.0

def _conf(sig):
    if not sig: return 0.0
    return float(sig.get("confidence", 0.0))

def _norm(sig, key, lo, hi):
    if not sig: return 0.5
    snap = sig.get("input_snapshot", {}) or {}
    val  = snap.get(key)
    if val is None: return 0.5
    return max(0.0, min(1.0, (float(val) - lo) / (hi - lo + 1e-9)))

def _norm_fin(sig, key, lo, hi):
    if not sig: return 0.5
    snap = sig.get("input_snapshot", {}) or {}
    val  = snap.get(key)
    if val is None: return 0.5
    try: return max(0.0, min(1.0, (float(val) - lo) / (hi - lo + 1e-9)))
    except: return 0.5

def _sign(sig, key):
    if not sig: return 0.5
    snap = sig.get("input_snapshot", {}) or {}
    val  = str(snap.get(key, ""))
    if not val: return 0.5
    return 1.0 if val.lstrip().startswith("+") or (val.replace("%","").replace(".","").lstrip("-").isdigit() and float(val.replace("%","")) > 0) else 0.0

def _portfolio_weight(approval):
    recs = approval.recommendations or []
    return float(recs[0].get("current_weight", 0.0)) if recs else 0.0

def _portfolio_total(approval):
    recs = approval.recommendations or []
    return sum(r.get("current_weight", 0.0) for r in recs)

def _find_signal(signals, agent_type):
    return next((s for s in signals if s.get("agent_type") == agent_type), None)
```

**特征向量定义（共 24 维）**：
```python
def build_embedding(approval, signals: list[dict]) -> list[float]:
    tech = _find_signal(signals, "technical")
    fund = _find_signal(signals, "fundamental")
    news = _find_signal(signals, "news")
    sent = _find_signal(signals, "sentiment")

    return [
        # 信号方向（4维）
        _dir_to_float(tech, "buy"),  _dir_to_float(tech, "sell"),
        _dir_to_float(fund, "buy"),  _dir_to_float(fund, "sell"),
        # 置信度（4维）
        _conf(tech), _conf(fund), _conf(news), _conf(sent),
        # 技术指标（6维）
        _norm(tech, "rsi",          0, 100),
        _norm(tech, "macd",        -5, 5),
        _norm(tech, "volume_ratio", 0, 5),
        1.0 if tech and "RSI" in tech.get("input_snapshot", {}).get("rule_triggered", "") else 0.0,
        1.0 if tech and "多头排列" in tech.get("input_snapshot", {}).get("rule_triggered", "") else 0.0,
        1.0 if tech and "空头排列" in tech.get("input_snapshot", {}).get("rule_triggered", "") else 0.0,
        # 基本面指标（4维）
        _norm_fin(fund, "pe_ratio",  0, 100),
        _norm_fin(fund, "pb_ratio",  0, 20),
        _norm_fin(fund, "roe",       0, 0.5),
        _sign(fund, "revenue_yoy"),
        # 情绪指标（2维）
        _conf(news),
        _conf(sent),
        # 组合上下文（4维）
        _portfolio_weight(approval),
        _portfolio_total(approval),
        1.0 if approval.approved else 0.0,
        float(len(approval.recommendations or [])),
    ]
```

**验收**：同一标的在技术面超卖条件下分析两次，两个节点的余弦相似度 > 0.80；完全不同标的不同条件下相似度 < 0.5。

**注入 Agent Prompt**（仅当有已结算案例时追加）：

在 `BaseAgent.complete()` 调用前，统一拼接历史案例段落：

```python
# agents/base.py LLMClient.complete 或各 Agent 调用处
def _build_history_hint(similar_cases: list) -> str:
    if not similar_cases:
        return ""
    lines = ["如有以下历史相似案例，请在摘要末尾补充一句参考说明（不超过 20 字）："]
    for i, c in enumerate(similar_cases[:3], 1):
        ret    = c.get("outcome_return", 0)
        dir_ok = "正确" if c.get("outcome_direction_correct") else "错误"
        ts     = str(c.get("timestamp", ""))[:10]
        dir_   = c.get("direction", "未知")
        lines.append(f"案例{i}（{ts}）：{dir_} 信号，实际收益 {ret:+.1%}，方向判断{dir_ok}")
    return "\n" + "\n".join(lines)
```

**验收**：
- 图谱有 5+ 条已结算案例后触发分析，Agent 推理摘要中出现历史案例参考描述。
- 无已结算案例时，prompt 中不追加历史段落，行为与 v1.0 相同。

#### 需求 3.2.4 图谱统计数据去随机化

**相似度趋势（similarity_trend）**：
按月统计该月所有节点两两之间的平均余弦相似度。节点数超过 50 时随机抽 50 对计算（避免 O(n²) 开销）。

**准确率（avg_accuracy）**：
仅统计 `outcome_settled = True` 的节点中 `outcome_direction_correct = True` 的比例，未结算节点排除。结果附带"已结算 N 笔"说明。

**图谱边（edge_count）**：
本版本定义：两节点 embedding 余弦相似度 > 0.7 则认为有边，`edge_count` 为满足此条件的节点对数量。节点数超过 100 时，随机抽样 100 个节点计算，避免性能问题。

**验收**：图谱页统计卡片"平均准确率"与实际结算记录中方向正确比例一致（误差 < 1%）。

### 4.2 功能需求

#### 需求 4.2.1 RL 环境定义

**新增文件**：`backend/services/rl_optimizer.py`

**状态空间（State，12维）**：
```python
state = np.array([
    w_tech, w_fund, w_news, w_sent,          # 当前 4 个 Agent 权重
    acc_tech, acc_fund, acc_news, acc_sent,   # 近 20 笔各 Agent 准确率（direction_correct 比例）
    market_volatility,                        # 近 20 日结算收益率标准差（0~1 归一化）

    avg_confidence,                           # 全局平均置信度

```

**动作空间（Action，离散625个）**：
每个 Agent 权重调整量 ∈ `{-0.05, -0.02, 0.0, +0.02, +0.05}`，共 5^4 = 625 个动作。

**动作解码方式**：
```python
DELTAS = [-0.05, -0.02, 0.0, 0.02, 0.05]

def decode_action(action_idx: int) -> dict:
    """将 0~624 的整数动作解码为 4 个 Agent 的权重调整量"""
    agents = ["technical", "fundamental", "news", "sentiment"]
    deltas = {}
    idx = action_idx
    for agent in reversed(agents):
        deltas[agent] = DELTAS[idx % 5]
        idx //= 5
    return deltas

def apply_action(current_weights: dict, action_idx: int) -> dict:
    """应用动作，归一化并钳制到 [0.05, 0.45]"""
    deltas = decode_action(action_idx)
    new_weights = {k: current_weights[k] + deltas[k] for k in current_weights}
    # 钳制
    new_weights = {k: max(0.05, min(0.45, v)) for k, v in new_weights.items()}
    # 归一化为和为 1.0
    total = sum(new_weights.values())
    return {k: round(v / total, 4) for k, v in new_weights.items()}
```

**奖励函数**：
```python
def compute_reward(settled_nodes: list) -> float:
    if not settled_nodes:
        return 0.0
    returns = [n.outcome_return for n in settled_nodes if n.outcome_return is not None]
    if not returns:
        return 0.0
    avg_return   = np.mean(returns)
    max_drawdown = min(returns)  # 简化：最差单笔收益作为回撤代理
    dir_correct  = np.mean([1.0 if n.outcome_direction_correct else 0.0 for n in settled_nodes])
    weights      = [n.embedding[0] if n.embedding else 0.25 for n in settled_nodes]  # 近似最大权重
    max_weight   = max(weights) if weights else 0.25
    conc_penalty = max(0.0, max_weight - 0.4) * 3.0

    reward = (
        avg_return   * 10.0
        + max_drawdown * 5.0    # max_drawdown 为负值，乘以 5.0 即为惩罚
        + dir_correct * 0.5
        - conc_penalty
    )
    return float(reward)
```

**环境实现**：继承 `gymnasium.Env`，`step()` 从历史决策数据库中按时序回放，不实时调用 AKShare。

```python
async def train_incremental(new_settled_nodes: list):
    from pathlib import Path
    import shutil

    model_path = Path("rl_models/rl_model.zip")
    model_path.parent.mkdir(exist_ok=True)

    env = AgentWeightEnv(load_all_settled_decisions())

    if model_path.exists():
        # 训练前备份
        ts = datetime.now().strftime("%Y%m%d_%H%M%S")
        shutil.copy(model_path, f"rl_models/{ts}_backup.zip")
        model = PPO.load(str(model_path), env=env)
    else:
        model = PPO("MlpPolicy", env, verbose=0, learning_rate=3e-4, n_steps=256)

    model.learn(total_timesteps=len(new_settled_nodes) * 50)
    model.save(str(model_path))

    # 计算新权重建议
    obs, _ = env.reset()
    new_weights = compute_weights_from_policy(model, obs)
    return new_weights

def compute_weights_from_policy(model, obs) -> dict:
    """从 PPO 策略输出推断最优权重"""
    agents = ["technical", "fundamental", "news", "sentiment"]
    # 从当前状态出发，贪婪地选择最优动作
    action, _ = model.predict(obs, deterministic=True)
    # 从 obs 中提取当前权重（状态向量前4维）
    current = {a: float(obs[i]) for i, a in enumerate(agents)}
    return apply_action(current, int(action))
```

**安全约束**（RL 输出必须通过统计约束过滤）：
```python
def apply_safety_constraints(rl_weights: dict, accuracy: dict) -> dict:
    """低准确率的 Agent 权重强制上限"""
    safe = dict(rl_weights)
    for agent, acc in accuracy.items():
        if acc < 0.45:
            safe[agent] = min(safe[agent], 0.10)
    # 重新归一化
    total = sum(safe.values())
    return {k: round(v / total, 4) for k, v in safe.items()}
```

### 4.3 验收标准
