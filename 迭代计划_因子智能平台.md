## 二、两个开源项目的核心思想借鉴

### 2.1 karpathy/autoresearch — 自主研究循环

**核心思想**：给 AI Agent 一个固定的实验预算和评估指标，让它在循环中不断尝试→评估→保留/丢弃，
每小时可完成 12 次独立实验，无需人工干预。

**迁移到量化平台的方式**：

| AutoResearch 概念 | 量化平台对应 |
|-------------------|------------|
| train.py（可修改的研究空间） | 因子表达式（可由 LLM 生成和变异） |
| prepare.py（固定基础设施） | 特征平台（固定，不参与实验） |
| val_bpb 评估指标 | IC_IR（因子信息比率，与词表无关的客观指标） |
| 5分钟训练预算 | N天回测窗口（固定计算预算） |
| keep/discard 决策 | 因子入库/淘汰决策 |
| program.md 研究方向 | 研究员输入的因子假设描述 |

**关键借鉴**：
- 每次实验结果立即评估，好的保留进因子库，差的丢弃并记录原因
- 固定计算预算（比如最多回测 60 个交易日）防止过度拟合
- Agent 可以在前人基础上变异，而不是每次从零开始

### 2.2 QuantaAlpha — LLM 驱动因子挖掘

**核心思想**：用自然语言描述研究方向 → LLM 生成多样化因子假设 → 代码化 → 回测验证 → 进化优化。
已验证在 CSI 300 上实现 27.75% 年化超额收益，Calmar 比率 3.48。

**迁移到量化平台的方式**：

| QuantaAlpha 模块 | 量化平台对应实现 |
|-----------------|---------------|
| llm（因子假设生成） | Agent 的 Prompt 工程，输出结构化因子表达式 |
| coder（代码生成） | Python 因子计算函数自动生成与沙盒执行 |
| backtest（回测验证） | 接入现有 backtest 引擎 + 真实 AKShare 数据 |
| pipeline（流程编排） | 现有 pipeline.py 扩展，增加因子查询步骤 |
| 进化算法 | 基于 IC 排名的选择→变异→淘汰循环 |

**关键借鉴**：
- IC（信息系数）作为因子质量的核心指标，与收益方向无关，不容易被噪声欺骗
- 多样性假设生成（diversified hypotheses）：避免所有因子都相关，形成有效组合
- 独立验证集防止过拟合：训练期做挖掘，验证期做评估，两者时间不重叠

### v9.3 IC 评估引擎

**核心公式**：
```
IC(t) = corr( factor_value(t), return(t→t+N) )
```
- `factor_value(t)`：截面日 t 各股票的因子值（已标准化）
- `return(t→t+N)`：未来 N 日（通常 5/10/20 日）的真实涨跌幅
- `corr`：Spearman 秩相关（对异常值更鲁棒）

**关键指标计算**（`backend/services/factor_eval.py`）：

```python
def evaluate_factor(
    factor_values: pd.Series,  # index=symbol, value=factor_value
    future_returns: pd.Series, # index=symbol, value=N日收益率
) -> dict:
    ic = factor_values.corr(future_returns, method='spearman')
    return {"ic": ic}

def evaluate_factor_series(
    factor_ts: pd.DataFrame,   # columns=date, rows=symbol
    return_ts: pd.DataFrame,   # 对齐后的未来收益
    forward_days: int = 5,
) -> dict:
    ics = []
    for date in factor_ts.columns:
        fv = factor_ts[date].dropna()
        fr = return_ts.get(date, pd.Series()).reindex(fv.index).dropna()
        common = fv.index.intersection(fr.index)
        if len(common) < 20:  # 覆盖股票数不足时跳过
            continue
        ic = fv[common].corr(fr[common], method='spearman')
        ics.append(ic)

    ics = pd.Series(ics)
    return {
        "ic_mean":         ics.mean(),           # 目标: > 0.03
        "ic_ir":           ics.mean() / ics.std(), # 目标: > 0.5
        "ic_positive_pct": (ics > 0).mean(),     # 目标: > 55%
        "ic_series":       ics.tolist(),
    }
```

**数据防泄漏规则**（与 v3.0 回测相同原则）：
- 计算因子值时只能用截面日 T 的数据
- 评估用的未来收益必须是 T+1 开盘之后的真实收益
- 训练期（挖掘）和验证期（评估）时间不重叠，建议 7:3 分割

**验收**：对 RSI_14 因子，在过去 60 个交易日数据上计算 IC 序列，
结果与手动 Excel 计算一致（偏差 < 0.001）。

### v10.1 LLM 因子假设生成器

研究员输入（自然语言）：
```

直觉：当融资比例过高时，市场杠杆过重，容易引发踩踏；过低时可能意味着情绪极度悲观。
建议探索：融资比例的绝对值、相对于历史分位数的偏离、变化率等维度。
```

### v10.2 因子代码自动生成与沙盒执行

**LLM 生成 Python 因子函数**：

```python
# 输入：因子假设
# 输出：可执行的 Python 函数

FACTOR_CODE_PROMPT = """
基于以下因子假设，生成一个 Python 函数来计算该因子：

假设：{hypothesis}
表达式：{expression}
依赖特征：{features_needed}

要求：
1. 函数签名: def compute_factor(df: pd.DataFrame) -> pd.Series
2. df 的列名对应 features_needed 中的特征名称
3. 返回 pd.Series，index 为 symbol，值为因子值
4. 包含异常处理（特征缺失时返回 pd.Series(dtype=float)）
5. 不得引入外部依赖（只用 pandas/numpy）
"""
```

**沙盒执行**（安全隔离）：
```python
async def execute_factor_code(code: str, feature_df: pd.DataFrame) -> pd.Series:
    """在受限环境中执行 LLM 生成的因子代码"""
    safe_globals = {
        "pd": pd,
        "np": np,
        "__builtins__": {
            "abs": abs, "min": min, "max": max, "len": len,
            "range": range, "list": list, "dict": dict,
        }
    }
    exec(code, safe_globals)
    compute_fn = safe_globals.get("compute_factor")
    if not compute_fn:
        raise ValueError("生成的代码未包含 compute_factor 函数")

    # 超时控制：因子计算不得超过 30 秒
    return await asyncio.wait_for(
        asyncio.to_thread(compute_fn, feature_df),
        timeout=30.0
    )
```

### v10.3 自主研究循环（核心）

**AutoResearch 迁移版**：固定计算预算内的自主实验循环

```
研究方向输入
     │
     ▼
生成 5~8 个因子假设（LLM，多样化）
     │
     ▼ 对每个假设
┌────▼──────────────────────────────────┐
│  1. 生成因子计算代码（LLM）            │
│  2. 计算截面因子值（沙盒执行）          │
│  3. 评估 IC_IR（固定回测窗口 60 交易日）│
│  4. 与现有因子库对比相关性（去重）      │
│  5. 判断：IC_IR > 0.5 且相关性 < 0.6  │
│     → 入库（active 状态）              │
│     → 否则丢弃，记录原因               │
└───────────────────────────────────────┘
     │
     ▼
进化优化轮（对 active 因子做变异）
  变异策略（随机选一种）：
  - 改变回看周期（5→10→20→60 日）
  - 添加波动率标准化（/ rolling_std(N)）
  - 替换为排名因子（pd.qcut → 分位数）
  - 加入行业中性化
  - 与另一个 active 因子线性组合

     │
     ▼ 变异后的因子再评估
重复直到：预算用完（最多 50 次迭代）或 IC_IR > 1.0（高质量停止）
```

**挖掘任务配置**：
```json
{
    "research_direction": "研究融资余额对短期价格的预测能力",
    "max_iterations": 50,
    "eval_window_days": 60,
    "forward_days": 5,
    "min_ic_ir_threshold": 0.5,
    "max_correlation_with_existing": 0.6,
    "universe": "A股全市场",
    "factor_types": ["sentiment", "technical"],
    "budget_minutes": 120
}
```

**验收**：
1. 输入一个研究方向，系统自动运行 10 次迭代（可在前端看进度）
2. 至少产生 1 个 IC_IR > 0.3 的因子进入因子库
3. 挖掘过程中的每次实验结果（假设、IC 值、结论）都可查看

### v10.4 因子健康度监控与自动维护

**因子衰减检测**：

因子发现后，每 20 个交易日重新计算最近 60 天的 IC_IR：

```python
# 衰减判断规则
if recent_ic_ir < 0.3 and original_ic_ir > 0.5:
    # 因子显著衰减
    factor.status = "degraded"
    trigger_remine(factor.research_direction)  # 触发重新挖掘

if recent_ic_ir < 0.1:
    # 因子失效
    factor.status = "archived"
```

### v11.3 因子驱动的持仓建议

**修改** `_make_recommendations`（`pipeline.py`）：
原来用随机偏移计算调仓幅度（`random.uniform(0.03, 0.08)`），
改为基于综合因子得分确定性计算：

```python
def _make_recommendations(signals, context):
    factor_context = context.get("factor_context", {})

    for sym in symbols:
        factor_score = factor_context.get(sym, {}).get("factor_composite_score", 0.5)
        agent_confidence = avg_conf  # 现有 Agent 平均置信度

        # 调仓幅度 = 因子得分偏离中性的程度 × Agent置信度
        factor_signal_strength = (factor_score - 0.5) * 2  # 映射到 [-1, 1]
        max_delta = 0.08  # 单次最大调仓幅度

        if final_dir == "buy":
            delta = max_delta * abs(factor_signal_strength) * agent_confidence
            rec = min(cur + delta, max_position_weight)
        elif final_dir == "sell":
            delta = max_delta * abs(factor_signal_strength) * agent_confidence
            rec = max(cur - delta, 0.0)
        else:
            rec = cur
```

**效果**：消灭 v7.0 技术债务中的随机数问题，调仓幅度有理可依。

**验收**：相同输入运行 5 次，推荐仓位完全一致（去除随机性）。

### v12.1 多因子选股回测

**策略逻辑**：

```python
# 每个调仓日（通常每月/每周）：
# 1. 计算所有股票的综合因子得分（IC_IR 加权）
# 2. 按得分排序，选取前 N% 构成多头组合，后 N% 构成空头组合
# 3. 等权持有至下个调仓日
# 4. 计算多头、空头、多空对冲的收益

composite_score[i] = Σ (factor_value[f,i] × IC_IR[f]) / Σ IC_IR[f]

# 多头组合：composite_score 前 20% 的股票
# 空头组合：composite_score 后 20% 的股票
```

## 十、明确不做的事

| 功能 | 不做的原因 |
|------|----------|
| Tick 级因子计算 | 日线级别因子够用，分钟级数据成本高且难获取 |
| GPU 加速训练 | 因子计算是 CPU 密集型，无需 GPU |
| 强化学习选因子 | IC 评估 + 进化算法更透明可解释，RL 黑盒风险高 |
| 海量因子库（>500个） | 超过 50 个高质量因子后边际价值递减，维护成本上升 |
| 高频因子（日内） | 平台定位是日频决策辅助，非高频交易系统 |
| Alpha 因子商业数据 | 优先用 AKShare 免费数据验证框架有效性 |

---

*核心迭代原则：先建因子基础设施（v9.0），再自主发现因子（v10.0），
再让 Agent 用上因子（v11.0），最后验证策略有效性（v12.0）。
不要在因子质量未验证前就让 Agent 使用，会把噪声放大成错误决策。*
