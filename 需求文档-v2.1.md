# spec: 001 — 分析任务状态持久化（离页不丢失）

**版本**：v1.0
**日期**：2026-03-31
**状态**：待审查
**优先级**：P0（用户体验阻断型缺陷）

---

## 1. 背景与问题

### 1.1 当前缺陷

用户在「触发分析」页面点击「开始分析」后，若切换至其他模块（如持仓、审批），再返回「触发分析」页时，**分析进度消失、结果丢失**，界面恢复为初始空状态。

### 1.2 根本原因（技术定位）

| 层次 | 文件 | 问题 |
|------|------|------|
| 前端状态 | `frontend/src/pages/AnalyzePage.vue:18-26` | `currentRun`、`pollTimer`、`isPolling` 全部存储在组件本地 `ref` 中，无持久化层 |
| 生命周期 | `AnalyzePage.vue:43` | `onUnmounted(stopPolling)` 在路由离开时清除所有计时器，正确清理资源，但同时丢失了运行状态 |
| 重挂载 | `AnalyzePage.vue:18` | 路由返回后组件重新挂载，`currentRun = null`，无法恢复正在进行中的任务 |
| 后端 | `backend/routers/decisions.py:173` | `asyncio.create_task(_background_pipeline(...))` 在服务器端独立运行，**不受前端导航影响**，任务继续正常完成 |

**结论**：后端分析从未中断；问题完全出在前端——没有持久化 `run.id`，导致重挂载后无法恢复轮询。

### 1.3 已有基础设施

- 后端已有 `websocket_manager.py`，`_background_pipeline` 执行过程中会调用 `ws_broadcaster` 推送事件（当前前端未使用）
- 后端 `GET /api/v1/decisions/{id}` 接口可随时查询任意 run 的最新状态
- 前端当前无 Pinia store，无全局状态管理

## 2. 目标

| 目标 | 描述 |
|------|------|
| 主目标 | 用户离开分析页再返回，**自动恢复**正在进行中的分析任务，看到实时进度 |
| 次目标 | 分析进行中时，在导航栏给出视觉提示，用户不需要记住去哪里查看 |
| 约束 | 不修改后端任何接口；不引入 WebSocket 重构（留给后续迭代）；改动范围最小化 |

## 3. 用户故事

### US-01：离页后恢复进度

> As a **量化分析员**,
> I want **离开触发分析页后再回来，分析还在继续显示进度**,
> So that **我可以在等待分析完成时自由浏览其他模块，不必守着分析页**。

### US-02：导航栏实时提示

> As a **量化分析员**,
> I want **分析进行中时，导航侧边栏的「触发分析」菜单项有明显的进行中标识**,
> So that **我在其他页面也能感知分析仍在运行，以及何时完成**。

### US-03：刷新页面后恢复

> As a **量化分析员**,
> I want **刷新浏览器后，如果分析未完成，页面自动恢复到分析中状态**,
> So that **意外刷新不会导致分析失联**。

### US-04：多标的并发安全

> As a **量化分析员**,
> I want **同时只允许存在一个活跃的分析任务（按钮在任务完成前保持禁用状态）**,
> So that **不会因误操作触发多个重复分析，浪费资源**。

## 4. 验收标准（EARS 格式）

### AC-01：正常路径 — 离页恢复

```
GIVEN 用户点击「开始分析」，分析任务状态为 running
WHEN 用户导航到其他任意页面后再返回「触发分析」页
THEN 进度面板自动展示，且继续实时更新（轮询恢复）
AND  已完成的 Agent 信号卡片全部显示，未完成的步骤显示 active 动画
AND  进度百分比与当前已完成 agent_signals 数量一致
```

### AC-02：正常路径 — 分析完成后返回

```
GIVEN 分析任务在用户离页期间已完成（status=completed）
WHEN 用户返回「触发分析」页
THEN 立即展示最终决策结果面板（方向、风险等级）
AND  「前往审批」按钮可用
AND  不再启动轮询（状态已终止）
```

### AC-03：导航栏提示

```
GIVEN 分析任务状态为 running
WHEN 用户在任意页面查看侧边导航
THEN 「触发分析」菜单项显示一个旋转动效图标或脉冲圆点
WHEN 任务完成（completed/failed）
THEN 该提示图标消失
```

### AC-04：刷新恢复

```
GIVEN 用户触发了分析，run.id 已持久化
WHEN 用户刷新浏览器（F5）
THEN 页面重新加载后自动恢复分析状态，继续轮询
AND  已有的 agent_signals 全部重新展示
```

### AC-05：任务完成后重置状态

```
GIVEN 分析任务已 completed 或 failed
WHEN 用户点击「重新分析」或修改标的后点击「开始分析」
THEN 清除旧的持久化状态，开始新任务
AND  旧的 agent_signals 面板清空
```

### AC-06：异常路径 — run.id 失效

```
GIVEN sessionStorage 中保存了一个旧的 run.id
WHEN 后端返回 404（如数据库重置）
THEN 前端静默清除 sessionStorage 中的 run.id
AND  页面显示空状态，不显示错误提示
```

### AC-07：并发保护

```
GIVEN 当前有 running 状态的任务
WHEN 用户在触发分析页面尝试点击「开始分析」
THEN 按钮保持禁用（loading 状态），无法触发新任务
AND  状态文字显示 "分析中... Xs"（计时器继续）
```

| 类别 | 要求 |

| 兼容性 | 不新增任何 npm 依赖；Pinia 已在 `package.json` 中声明则使用，否则用 `sessionStorage` 直接实现 |
| 代码范围 | 改动文件 ≤ 3 个；不修改后端任何文件 |
| 回归安全 | 现有轮询逻辑（`startPolling` / `stopPolling`）行为不变，只增加持久化包装 |

## 7. 技术方案（供设计参考）

### 7.1 推荐方案：Pinia Store + sessionStorage 双保险

```typescript
// frontend/src/stores/analysisStore.ts（新增）
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { DecisionRun } from '@/types/decision'

export const useAnalysisStore = defineStore('analysis', () => {
  const activeRunId   = ref<string | null>(sessionStorage.getItem('activeRunId'))
  const currentRun    = ref<DecisionRun | null>(null)
  const isPolling     = ref(false)

  const isRunning = computed(
    () => isPolling.value || currentRun.value?.status === 'running'
  )

  function setActiveRun(run: DecisionRun) {
    currentRun.value = run
    activeRunId.value = run.id
    sessionStorage.setItem('activeRunId', run.id)
  }

  function clearActiveRun() {
    currentRun.value = null
    activeRunId.value = null
    sessionStorage.removeItem('activeRunId')
  }

  return { activeRunId, currentRun, isPolling, isRunning, setActiveRun, clearActiveRun }
})
```

### 7.2 AnalyzePage.vue 改动要点

- `currentRun` / `isPolling` 从 store 读取，不再是本地 `ref`
- `onMounted`：若 `store.activeRunId` 存在且 `store.currentRun?.status === 'running'`，立即调用 `startPolling(store.activeRunId)` 恢复轮询
- `handleTrigger` 成功后调用 `store.setActiveRun(run)` 替换原来的 `currentRun.value = run`
- `onUnmounted`：只停计时器，不清 store（改变原来 `stopPolling` 的副作用）

### 7.3 导航栏改动

- 侧边栏组件（`AppLayout` 或 `Sidebar`）通过 `useAnalysisStore().isRunning` 判断是否显示脉冲圆点
